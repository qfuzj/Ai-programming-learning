# campus-resource-sharing 后端接口规范文档

## 1. 文档说明

本文档定义校园闲置物资共享平台后端接口规范，供后端开发、前端联调、AI代码生成统一遵循。

- 项目名：campus-resource-sharing
- 后端技术栈：Spring Boot 3 + MyBatis-Plus + MySQL + JWT
- 接口风格：RESTful
- 返回格式：统一 Result 结构
- 字符编码：UTF-8
- 数据格式：JSON

---

## 2. 统一接口规范

## 2.1 请求头
```http
Content-Type: application/json
Authorization: Bearer <token>
```

## 2.2 统一返回结构
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 字段说明
- code：状态码
- message：提示信息
- data：业务数据

## 2.3 通用状态码

| code | 含义 |
|---|---|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或token失效 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 500 | 系统内部错误 |

}

## 14. 安全与约束

1. 所有需要登录的接口必须校验 JWT
2. 所有用户私有资源必须校验归属关系
3. 管理员接口必须校验角色
4. 文件上传限制格式与大小
5. 所有新增/修改接口必须进行参数校验
6. 收藏、下单、评价等操作应避免重复提交
7. 商品删除建议采用逻辑删除
8. 接口命名和字段命名必须前后端统一

---

## 15. AI代码生成要求

后续 AI 生成后端代码时必须遵守：
- 使用 Spring Boot 3
- 使用 MyBatis-Plus
- 使用统一 `Result<T>` 返回
- 使用 `@RestControllerAdvice` 全局异常处理
- 使用 `jakarta.validation` 进行参数校验
- 使用 JWT 实现登录鉴权
- Controller、Service、Mapper、Entity、DTO、VO 分层清晰
- 所有接口优先按本文档实现
