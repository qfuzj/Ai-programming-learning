## 1. 文档说明

### 1.1 适用范围
本文件基于当前前端工程实际状态更新，目标是保证文档与代码一致，并指导后续联调与扩展。

### 1.2 当前前端技术栈（实际）
- Vue 3
- Vite
- TypeScript（strict）
- Element Plus（自动按需导入）
- Pinia
- Vue Router
- Axios

说明：
- 历史文档中出现的 Naive UI、vicons、复杂后台模板能力（租户/动态菜单/字典中心）已不再作为当前基线。

### 1.3 当前设计目标
- 用户前台与管理后台路由、布局、权限完全隔离。
- 权限模型简化为双角色：ADMIN / USER。
- 先建立可运行骨架，再逐页接入后端接口。

## 4. 路由守卫规则（当前实现）

文件：`router/guards/auth.ts`

- 未登录访问受保护页面：
  - 访问 `/admin/**` 跳转 `/admin/login`
  - 访问用户端受保护路由跳转 `/login`
- 已登录访问登录页：
  - 访问 `/login`：ADMIN 跳转 `/admin/dashboard`，USER 跳转 `/`
  - 访问 `/admin/login`：ADMIN 跳转 `/admin/dashboard`，USER 跳转 `/403`
- 已登录但角色不匹配：
  - 统一跳转 `/403`
- 鉴权判断：
  - 通过 `to.matched` 聚合 `requiresAuth` 与 `roles`，父子路由 meta 统一生效

## 7. API 模块（当前实现）

### 7.1 通用封装
文件：`api/http.ts`

- 请求拦截：自动注入 Bearer token
- 响应拦截：处理统一响应结构 `{ code, message, data }`
- 异常处理：统一消息提示

### 7.2 已拆分业务模块
- `api/auth.ts`
- `api/admin-audit.ts`
- `api/admin-config.ts`
- `api/admin-log.ts`
- `api/common.ts`
- `api/scenic.ts`
- `api/ai.ts`
- `api/chat.ts`
- `api/dict.ts`
- `api/favorite.ts`
- `api/file.ts`
- `api/history.ts`
- `api/profile.ts`
- `api/review.ts`
- `api/user.ts`
- `api/itinerary.ts`
- `api/dashboard.ts`

### 7.3 模块职责说明
- `auth.ts`：登录、登出、刷新 Token、验证码、注册、重置密码；按登录入口映射前端角色（`loginAdmin`→ADMIN，`loginUser`→USER）。
- `auth.ts` 负责认证闭环和验证码。登录响应中 `roleCode` 映射到前端 `role` (ADMIN/USER) 的逻辑由 `normalizeLoginResponse` 处理；特别地，`loginAdmin` 接口响应中若 `roleCode` 为空则使用 `defaultRole` 以保证管理端登录稳定性。
<!--  -->
- `common.ts`：地区树、标签查询等公共基础数据。
- `scenic.ts`：用户端景点查询与管理端景点列表查询。
- `profile.ts`：个人中心资料、画像、偏好标签。
- `user.ts`：管理端用户分页、详情、状态更新。
- `chat.ts`：会话列表、会话详情、消息列表、发送消息、删除会话；会话时间优先使用 `lastMessageAt` 映射到 `updatedAt`。
- `ai.ts`：推荐首页与 AI 对话便捷封装，内部复用 `chat.ts`。
- `favorite.ts`：收藏列表与收藏/取消收藏。
- `history.ts`：浏览历史上报、查询、删除、清空。
- `review.ts`：我的点评、景点评价列表、发布与删除点评。
- `itinerary.ts`：行程 CRUD、行程项管理、AI 行程生成。
- `dashboard.ts`：管理端工作台概览、用户增长趋势、景点热度排行、推荐分析、LLM 分析。
- `admin-audit.ts`：管理端审核列表与通过/拒绝/隐藏操作。
- `admin-config.ts`：管理端系统配置分页查询、配置详情与更新。
- `admin-log.ts`：管理端操作日志分页查询与详情。
- `dict.ts`：状态/类型字典查询（审核状态、会话类型、内容类型等）。
- `file.ts`：上传凭证、上传回调、文件资源查询与删除。

## 11. 组件设计与交互

### 11.1 景点详情页 (ScenicDetailView.vue)

当前景点详情页补充了点评维度的扩展能力，并引入了独立的子组件抽离逻辑。

- **子组件：`ReviewItemCard.vue`**
  渲染点评项，承担独立的点赞、回复等交互动作，方便复用于其他需要显示点评的页面。

- **交互动作说明**
  - **展示层**：加载点评数据列表，渲染多个 `ReviewItemCard.vue` 实例。
  - **点评点赞（Toggle Like）**：
    - 点击时发起点赞 / 取消点赞请求。
    - 前端无需等待重新获取列表数据即可 optimistic 地切换高亮显示和数量 (`likeCount`)。
  - **回复点评**：
    - 展示回复的数量 (`replyCount`) 与展开按钮。
    - 展开回复列表：首次点击加载 `GET /api/user/reviews/{reviewId}/replies`，显示所有关于本条点评的跟帖。
    - 提交新回复：在展开框底部或弹窗显示 Input 框输入，确认并 `POST /api/user/reviews/{reviewId}/reply` 成功后、清空文本框，重新获取当前回复列表以刷新组件视图。
