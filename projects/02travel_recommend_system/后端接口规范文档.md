# 后端接口规范文档

## 1. 文档说明

### 1.1 项目名称
LLM-Travel-Advisor

### 1.2 技术栈
- Spring Boot 3
- MyBatis-Plus
- MySQL
- Redis
- JWT

### 1.3 接口设计原则
- 前后端分离
- RESTful 风格
- 用户端与管理端接口分离
- 使用 JWT 做身份认证
- 使用 Redis 存储验证码、Token、黑名单与推荐缓存
- 使用统一响应体、统一异常码、统一分页规范
- 文件模块仅保留接口边界，不做 MinIO 深实现

### 1.4 接口前缀约定
- 用户端接口前缀：`/api/user`
- 管理端接口前缀：`/api/admin`
- 公共接口前缀：`/api/common`

## 2. 统一规范

### 2.1 统一响应结构

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1710000000000,
  "requestId": "trace-uuid"
}
```

### 2.2 分页响应结构

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [],
    "total": 100,
    "pageNum": 1,
    "pageSize": 10,
    "totalPage": 10
  },
  "timestamp": 1710000000000,
  "requestId": "trace-uuid"
}
```

### 2.3 通用请求头
| Header        | 是否必须      | 说明             |
| ------------- | ------------- | ---------------- |
| Authorization | 否/按接口要求 | Bearer Token     |
| Content-Type  | 是            | application/json |
| X-Request-Id  | 否            | 请求链路追踪ID   |

### 2.4 通用分页参数
| 参数      | 类型    | 必填 | 说明             |
| --------- | ------- | ---- | ---------------- |
| pageNum   | Integer | 否   | 页码，默认1      |
| pageSize  | Integer | 否   | 每页条数，默认10 |
| keyword   | String  | 否   | 关键词           |
| sortBy    | String  | 否   | 排序字段         |
| sortOrder | String  | 否   | asc / desc       |

### 2.5 统一异常码建议
| code | 含义              |
| ---- | ----------------- |
| 200  | 成功              |
| 400  | 参数错误          |
| 401  | 未登录或Token失效 |
| 403  | 无权限            |
| 404  | 资源不存在        |
| 409  | 数据冲突          |
| 422  | 业务校验失败      |
| 429  | 请求过于频繁      |
| 500  | 系统异常          |
| 502  | LLM服务调用失败   |

## 18. 文件资源模块（预留）

### 18.1 设计原则
- 本阶段仅保留文件资源元数据管理接口
- 文件上传服务抽象为 FileService

## 19. 安全认证设计说明

### 19.1 JWT策略
- accessToken：短期有效，例如2小时
- refreshToken：长期有效，例如7天
- token中包含 userId / roleType / loginType / tokenVersion

### 19.2 Redis Key设计建议
- `auth:captcha:{captchaId}`
- `auth:user:token:{userId}:{tokenId}`
- `auth:admin:token:{adminId}:{tokenId}`
- `auth:blacklist:{tokenId}`
- `recommend:user:{userId}:home`
- `recommend:scenic:{scenicId}:similar`
- `chat:summary:{conversationId}`

### 19.3 黑名单机制
- 登出后 accessToken 加入黑名单
- 修改密码后旧 token 批量失效
- 用户被禁用时立即清理 token

### 19.4 越权防控
- 用户数据必须按 userId 隔离
- 管理接口必须校验管理员身份
- 查询他人行程、会话、收藏等需强制拦截
- 审核、配置、景点管理仅管理员可操作
