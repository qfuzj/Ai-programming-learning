# 后端接口规范文档

## 1. 文档说明

### 1.1 项目名称
LLM-Travel-Advisor

### 1.2 当前后端技术栈
- Spring Boot 3
- MyBatis-Plus
- MySQL
- Redis
- JWT

### 1.3 接口分层与前缀
- 用户端：`/api/user/**`
- 管理端：`/api/admin/**`
- 公共接口：`/api/common/**`
- 字典接口：`/dict/**`、`/api/dict/**`

说明：
- 字典接口当前兼容 `/dict` 与 `/api/dict` 两套前缀，便于和现有前端保持一致。
- 本文以当前控制器代码为准，不再保留历史设计稿中的未落地接口。

## 2. 统一规范

### 2.1 统一响应结构

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": "2026-04-17T10:00:00",
  "requestId": "trace-id"
}
```

说明：
- 前端 `axios` 响应拦截器会直接解包 `data` 字段。
- 业务成功码使用 `200`。

### 2.2 分页结构

```json
{
  "records": [],
  "total": 100,
  "pageNum": 1,
  "pageSize": 10,
  "totalPage": 10
}
```

### 2.3 通用分页参数
| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `pageNum` | Integer | 默认 `1`，最小 `1` |
| `pageSize` | Integer | 默认 `10`，范围 `1~200` |
| `keyword` | String | 可选，模糊搜索 |
| `sortBy` | String | 可选，具体支持值由模块定义 |
| `sortOrder` | String | 可选，`ASC` / `DESC` |

### 2.4 通用认证头
| Header | 说明 |
| --- | --- |
| `Authorization` | `Bearer <token>` |
| `Content-Type` | `application/json` |

### 2.5 常用状态/枚举约定
- 用户角色：前端只区分 `USER` / `ADMIN`
- 景点状态：`0` 下架，`1` 上架，部分说明中出现 `2` 审核中
- 是否推荐：`0` 否，`1` 是
- 是否公开：`0` 否，`1` 是
- 会话类型：`1` 智能客服，`2` 行程规划，`3` 景点咨询
- 会话状态：`0` 关闭，`1` 活跃
- 消息内容类型：`1` 文本，`2` 图片

## 10. 推荐与聊天模块

### 10.1 推荐接口
| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/user/recommend/home` | 首页推荐分页 |
| `GET` | `/api/user/recommend/scenic/{scenicId}` | 相似景点推荐 |
| `POST` | `/api/user/recommend/feedback/exposure` | 曝光反馈 |
| `POST` | `/api/user/recommend/feedback/click` | 点击反馈 |
| `POST` | `/api/user/recommend/feedback/favorite` | 收藏反馈 |

推荐项结构：
```json
{
  "recommendRecordId": 1,
  "resultItemId": 10,
  "scenicId": 1001,
  "scenicName": "西湖",
  "coverImage": "http://...",
  "score": 4.6,
  "reason": "景点口碑稳定，且与您近期偏好的自然风光更匹配",
  "sourceType": "TAG",
  "rankScore": 1.82
}
```

说明：
- `score` 是景点公共评分，来自用户点评平均分。
- `rankScore` 是当前推荐链路的规则排序分。
- `reason` 优先由阿里 DashScope 批量生成；若 LLM 超时、异常或 JSON 解析失败，会降级为规则文案。
- 推荐记录会写入 `recommend_record.llm_used`、`recommend_record.llm_call_log_id`，LLM 调用日志写入 `llm_call_log`，`call_type = recommend`。

## 13. 当前已知对齐问题

以下问题是基于当前代码扫描得到的“现状”，已写入文档方便后续联调：

1. 行程 AI 生成接口未实现  
   前端保留了 AI 行程生成页面与占位 API，但后端控制器没有对应接口。

2. 景点评分模型已收口  
   scenic 模块统一使用 `score` 表示景点公共展示分；原 `ratingScore` 已移出 scenic DTO / VO，个性化推荐分继续使用推荐接口中的 `rankScore`。
